# DB 품질 감사 보고서 — raw 원본 전수 대조 (2026-07)

품질 감사 2부(승인 계획의 후반부) 산출물. **raw/ 원본(공표 DOCX·PDF) ↔ 코퍼스(corpus_md/·guidelines_md/)의
전수 문단 대조**로, 지금까지 파이프라인의 어느 검증도 다루지 않았던 raw→md 변환 구간을 감사했다.
검색 품질 평가(접합 J2·J3)는 1부 절체 게이트에서 기수행 — §6에 결과만 통합 수록.

| 항목 | 내용 |
|---|---|
| 감사 시점 | 2026-07-12, 컬렉션 `standards_20250829_bgem3_v2` 절체 직후 |
| 대조 대상 | 코퍼스 **전 211파일 21,105문단** (kifrs 163 + ksa 39 + guide 9 — 보류 없음) |
| 도구 | `eval/audit_extract_raw.py`(독립 추출) + `eval/audit_compare.py`(2층 대조) |
| 자가검증 | 변조 3건 주입 → 3/3 탐지 (kifrs_1115 기준선 잡음 1건 대비 정확히 +3) |

## 1. 방법

**독립성 원칙**: 추출기는 피감 파이프라인(normalize_corpus.py·형제 파싱 프로젝트)의 로직을 재사용하지
않고 OOXML/PDF 명세만으로 구현했다. 같은 버그를 공유하면 대조가 무의미해지기 때문이다.

- **추출** — K-IFRS 63건+개념체계 3건: 문단번호가 리터럴 텍스트(numId=0으로 자동번호 해제)임을 실측 확인,
  본문·표·각주 텍스트 직추출. 감사기준 4건: 문단번호가 전부 numbering.xml 자동번호 → 전개기 구현
  (abstractNum 카운터 + startOverride 1,290건 재시작, 통합본은 style '10' 제목으로 36분할).
  실무지침 PDF 5건: PyMuPDF로 추출(2018-1·3은 난독화 폰트 코드포인트 +0x912A 복원 — pdfplumber/pdfminer는
  해당 글리프를 드롭하므로 사용 불가).
- **L1 존재성** — 코퍼스 각 문단의 물리 행을 정규화(NFKC·공백 전제거·인용부호/콜론/대괄호 접기) 후
  raw 전체 스트림에서 부분열 검색. 실패 행은 40자 조각 존재율로 OK/PARTIAL(≥0.5)/MISSING 버킷.
- **L2 번호 정렬** — 문서(base) 단위로 정본+BC+IE 갈래의 코퍼스 문단번호 합집합 ↔ raw 추출 번호의
  차집합. raw−corpus가 수록 누락 탐지의 핵심(L1은 코퍼스→raw 방향이라 통탈락 문단을 못 본다).
  갈래 합성 ID(IE사례N-x·IE부록X-n·BC-x)는 규약 4.3-5 문법을 역변환해 원번호로 복원 후 대조.

## 2. 결과 요약

```
L1: 21,105문단 중 OK 20,942 (99.23%) | PARTIAL 158 (0.75%) | MISSING 5 (0.02%)
L2 raw−corpus: 867건 — DO 186 · IN 137 · BCIN 20 · BC 518(cf/ps2 구역 514 + 개별 4) · 기타 6
```

MISSING 5건은 전건 원인 규명 완료(표 직렬화 2·도해 재정리 1·§3 F-5 관련 2), PARTIAL은
대부분 표 직렬화 순서 차이 잡음으로 판정 — §4. 진성 발견은 L2에서 나왔다 — §3.

## 3. 진성 발견

### F-1. K-IFRS BC 문단 4건 통탈락 (결함 — LEDGER C-15 등재 문안 §7)

raw DOCX에 실존하는 결론도출근거 문단이 ifrs_md(형제 프로젝트 산출)에서 텍스트째 탈락, 코퍼스·DB에 부재:

| 문단 | raw 원문 첫머리 | ifrs_md | corpus |
|---|---|---|---|
| 1016::BC35F | "IASB는 2007년에 발표된 IFRS 연차개선…" | 목차 표에만 흔적 | 없음 |
| 1028::BC20 | "IASB는 투자의 일부가 IAS 28의 문단 18에 따라…" | 없음 | 없음 |
| 1041::BC3 | "2008년 5월에 IFRS 연차개선…" | 없음 | 없음 |
| 1101::BC2 | "1998년에 발표된 국제회계기준해석서 제8호(SIC 8)…" | 없음 | 없음 |

전량이 BC 갈래라 정본 검색에는 무영향. 보수(원문 보완→재적재)는 별도 지시로.

### F-2. IN(도입)·DO(반대의견)·BCIN(BC 도입) 구역 전체 미수록 (수록 범위 미결정 — 343문단)

raw에는 있으나 ifrs_md 변환 단계에서 구역째 탈락해 코퍼스에 없다: IN 137(1105·1109·1110·1113·1116 등),
DO 186(반대의견), BCIN 20(1109 결론도출근거 도입부). 1부 BC·IE 확장은 ifrs_md에 존재하는 것을 전부
수록했으므로 1부 결함이 아니라 **상류(raw→md) 미변환**이다. 특히 BCIN은 bc 갈래 취지에 부합해
수록 가치가 있고, IN·DO는 별도 판단 사안.

### F-3. 개념체계(cf)·실무서2(ps2)의 BC 미수록 (수록 범위 미결정 — 약 514문단)

BC·IE 확장(1부)의 대상이 기준서 60건이어서 개념체계 문서 3종의 BC는 범위 밖이었다.
raw 실측: kifrs_cf BC0.x~BC8.x 계열 다수 + kifrs_ps2 BC1~BC59. (kifrs_mc는 BC 없음)

### F-4. 표 위첨자 각주의 수치 병합 (원본 유래 — 경미, 검색 영향 없음)

kifrs_1036_ie 등의 표에서 "300¹"이 "3001", "22,165¹"이 "22,1651"로 수록됨. **raw DOCX 텍스트
자체가 병합돼 있음을 확인** — 위첨자 서식 정보만으로 구분되는 원본 한계의 전파라 코퍼스 결함이 아니나,
표를 읽는 이용자에게 수치 왜곡으로 보일 수 있다. (예: 1036_ie 표의 "24,7251,2" = 24,725 + 각주 1,2)

### F-5. 실무지침 수작업 변환의 무언 교정·재구성 2건 (경미 — guidelines_md README 결정과의 정합 확인 사안)

- guide_2014-1 부칙 시행일: 원문(.doc) "2005년 12월 22**부터**"(오탈자)를 코퍼스가 "22**일**부터"로
  교정 — README 결정 7("원문 오탈자 그대로 유지")과 어긋나는 무언 교정.
- guide_2015-1 문단 1: 코퍼스 문장 구성이 원문과 다름(제정개요·전문 문구의 재구성으로 추정).
  guidelines_md는 수작업 산출물이라 의도된 편집일 수 있으나 README에 기록이 없다.

### 참고. raw DOCX 자체의 국소 결함 (코퍼스가 공표본과 일치 — 무결 방증)

- ISA 701 문단 5가 raw DOCX의 Word 번호 계산으로는 "4"(중복) — 코퍼스는 공표본대로 5. 원인: 문단 5가
  다른 abstractNum(608→abs192)을 참조하는 문서 저작 오류.
- 감사기준 통합본에 번호를 소비하는 빈 문단 존재(560 A8 자리 등) — 코퍼스 번호가 공표본과 일치.

## 6. 검색 품질 평가 (접합 J2·J3 — 1부에서 기수행, 결과 통합)

routing_gold 102조서 2트랙(R: 제목 원문 / C: 정답 누설 제거), `eval/audit_retrieval.py`:

| 지표 (macro) | v1 | v2 (현행) | v2+옵트인 |
|---|---|---|---|
| 트랙R recall@10 | 0.6845 | **0.7039** | 0.7060 |
| 트랙R recall@20 | 0.7470 | 0.7446 | 0.7523 |
| 트랙R MRR | 0.6487 | **0.6612** | 0.6478 |
| 트랙C recall@10 | 0.5752 | 0.5602 | 0.5232 |
| 트랙C MRR | 0.5792 | 0.5495 | 0.5273 |

- J2(절체 게이트): 트랙R 회귀 없음(+1.9pp) — 절체 승인 근거. 트랙C 소폭 하락은 sparse vocab 전체
  재계산 요동 범위로 판정(유일 신규 탈락 조서는 질의 자체가 퇴화된 사례).
- J3(옵트인 민감도): 전부 ON 시 recall@20 +0.8pp vs MRR·트랙C 하락 — **기본 옵트아웃 설계 확증**.

## 7. 처분 (2026-07-12 사용자 결정 반영)

1. **C-15 등재 승인** — F-1 4건을 LEDGER에 등재함.
2. **수록 범위 결정**: **BCIN 20문단 + 개념체계(cf) BC만 반영**하기로. IN·DO는 미수록 유지,
   ps2 BC는 보류. 반영은 별도 지시로(ifrs_md 재변환 또는 raw 직접 변환 → 재임베딩 → 재적재).
3. **F-4**: 원본 유래 — 기록만, 코퍼스 수정 없음.
4. **F-5**: guidelines_md README 결정사항 9로 기록 보완 완료(2026-07-13) — 두 건 모두 현행 유지.
5. **참고**: 개념체계(CF·MC·PS2)는 기본 검색에서 기준서 본문과 혼합됨(구분 축 없음, standard_no
   필터로만 분리 가능) — cf BC 반영 작업 때 구분 축 설계를 함께 검토.
